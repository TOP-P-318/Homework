# Домашнее задание 2: Фильтрация заметок

Сейчас приложение умеет отображать список заметок, добавлять новые записи, удалять их и менять признак важности. Однако
пользователю приходится видеть все заметки сразу, даже если ему нужны только важные или только обычные записи.

В этом задании необходимо добавить фильтрацию заметок по полю `IsImportant`. Фильтр должен работать на стороне сервера и
сохраняться после действий с заметками.

## 1.1 Добавление фильтра на страницу со списком заметок

### 1.1.1 Добавить форму с `<select>` для выбора режима фильтрации

На страницу со списком заметок необходимо добавить отдельную форму с методом `GET`. Внутри формы должен быть элемент
`<select name="filter">`, который передает выбранное значение фильтра в обработчик страницы.

```HTML
<!-- File: Pages/Index.cshtml -->
<form method="get">
    <select name="filter">
        <!-- TODO: добавить option для всех режимов фильтрации -->
    </select>

    <button type="submit">Применить</button>
</form>
```

Форму можно расположить над списком заметок, чтобы пользователь сразу видел, какой режим отображения сейчас доступен.

[ ВАЖНО ] Фильтрация должна выполняться через `GET`, а не через `POST`, потому что выбор фильтра не изменяет данные на
сервере, а только меняет способ отображения списка.

### 1.1.2 Добавить enum для описания режимов фильтрации

Чтобы не передавать фильтр обычной строкой, необходимо завести отдельный `enum`.

```C#
// New file: Models/ImportanceFilter.cs
public enum ImportanceFilter
{
    All,
    ImportantOnly,
    NotImportantOnly
}
```

Поведение каждого значения:

- `All` - отображаются все заметки;
- `ImportantOnly` - отображаются только важные заметки;
- `NotImportantOnly` - отображаются только неважные заметки.

## 1.2 Обновление интерфейса сервиса заметок

### 1.2.1 Изменить метод `GetAll`

Метод получения всех заметок должен принимать параметр фильтра.

```C#
// File: Services/INoteService.cs
public interface INoteService
{
    List<Note> GetAll(ImportanceFilter filter = ImportanceFilter.All);
}
```

Значение по умолчанию нужно для того, чтобы вызов `GetAll()` без параметров продолжал возвращать все заметки.

### 1.2.2 Обновить все места, где вызывается `GetAll`

После изменения интерфейса необходимо проверить все классы, где вызывается метод `GetAll`. В обработчике страницы нужно
передавать туда текущий фильтр.

```C#
Notes = _notes.GetAll(filter);
```

Если где-то список заметок должен отображаться без фильтрации, можно оставить вызов без параметров.

## 1.3 Реализация фильтрации в `InMemoryNoteService`

### 1.3.1 Добавить обработку `ImportanceFilter`

В сервисе, который хранит заметки в памяти, можно фильтровать обычный список через `Where`.

```C#
// File: Services/InMemoryNoteService.cs
public List<Note> GetAll(/* TODO: добавить параметр фильтра */)
{
    // ...
    
    _notes.Where(
        // TODO: если выбраны только важные заметки - вернуть только записи с IsImportant == true
        // TODO: если выбраны только неважные заметки - вернуть только записи с IsImportant == false
        // TODO: если выбран режим All - вернуть все заметки
    );
    
    // ...

}
```

Такой подход, при определении принадлежности элемента к фильтру внутри `Where` позволяет нам написать более компактный
код, чем если выполнять 3 различные flow в зависимости от значения `filter`.

[ ВАЖНО ] Не нужно удалять элементы из исходного списка. Фильтр должен только выбирать, какие заметки показать на
странице.

## 1.4 Реализация фильтрации в `SqliteNoteService`

### 1.4.1 Построить запрос через `IQueryable`

В сервисе, который работает с базой данных, фильтрацию лучше применять на уровне SQL-запроса, а не после загрузки всех
заметок в память.

```C#
// File: Services/SqliteNoteService.cs
public List<Note> GetAll(/* TODO: добавить параметр фильтра */)
{
    var query = _context.Notes.AsQueryable();

    switch (filter) 
    {
        // TODO: если выбраны только важные заметки - добавить Where
        // TODO: если выбраны только неважные заметки - добавить Where
        // TODO: если выбран режим All - не добавлять дополнительный Where
    }

    // TODO: отсортировать, преобразовать к Note и вернуть список
}
```

Здесь мы в каждой `case`-ветке присваиваем

```C#
query = query.Where(/*...*/)
```

Таким образом мы разделяем состав запроса к БД в зависимости от условия фильтра (в случае, когда `filter == All`,
присваивать и вовсе ничего не придется). Мы делаем это, чтобы запрос в базу данных всегда был минималистичным и не
перегруженным. При работе с `InMemoryNoteService` мы о таком не заморачивались, так как там вся обработка выполняется в
оперативной памяти, и она достаточно быстрая.

## 1.5 Сохранение фильтра после действий с заметками

### 1.5.1 Добавить параметр `filter` во все обработчики

Если пользователь выбрал фильтр, а потом добавил заметку, удалил ее или поменял важность, фильтр не должен сбрасываться.
Для этого во все обработчики, которые возвращают пользователя обратно на `Index`, нужно добавить параметр `filter`.

```C#
// File: Pages/Index.cshtml.cs
public IActionResult OnPostDelete(int id, /* TODO: добавить параметр filter */)
{
    _notes.Delete(id);

    return RedirectToPage(/* TODO: передать filter */);
}
```

Аналогично нужно обработать остальные действия:

- добавление новой заметки;
- удаление заметки;
- изменение важности;
- обработку ошибок, если в проекте есть отдельный обработчик `OnFail` или похожий метод.

### 1.5.2 Добавить скрытое поле во все формы заметок

Форма фильтрации передает `filter` через `GET`, но остальные формы сами по себе об этом параметре не знают. Поэтому в
каждую форму, которая выполняет действие с заметкой, нужно добавить скрытое поле.

```HTML
<!-- File: Pages/Index.cshtml -->
<input type="hidden" name="filter" value="@Model.Filter"/>
```

[ ВАЖНО ] Это поле не нужно добавлять в саму форму фильтрации. Оно нужно только в формы, которые выполняют действия над
заметками.

### 1.5.3 Передавать фильтр в `RedirectToPage`

После `POST`-запроса необходимо возвращаться на страницу через `RedirectToPage`, передавая текущий фильтр.

```C#
return RedirectToPage(/* TODO: передать filter */);
```

Если этого не сделать, после любого действия пользователь снова окажется в режиме `All`.

## 1.6 Подводные камни

### 1.6.1 Фильтр и перестановка заметок

Если в проекте есть перестановка заметок через `drag-and-drop`, фильтрация может сделать ее поведение странным.
Например, пользователь видит только важные заметки, но в общем списке между ними могут находиться еще и неважные.

Для этого домашнего задания можно выбрать один из вариантов:

- не исправлять `drag-and-drop`, если он не требуется в вашей версии проекта;
- отключить перетаскивание при `filter != ImportanceFilter.All`;
- учитывать фильтр при отправке нового порядка заметок, если хотите реализовать более аккуратное поведение.

## Требования

- На странице есть форма фильтрации с элементом `<select>`.
- Доступны три режима фильтрации: все заметки, только важные, только неважные.
- Фильтрация выполняется на сервере через `INoteService.GetAll(filter)`.
- При выборе фильтра на странице отображаются только подходящие заметки.
- Выбранный фильтр остается выбранным после перезагрузки страницы.
- При добавлении, удалении и изменении важности заметки фильтр не сбрасывается.
- Все формы, которые выполняют действия над заметками, передают текущий `filter`.
- `InMemoryNoteService` фильтрует заметки через LINQ по списку.
- `SqliteNoteService` фильтрует заметки через `IQueryable`, добавляя `Where` только при необходимости.
- Для режима `All` возвращаются все заметки без дополнительной фильтрации.
